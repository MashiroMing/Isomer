# MolGenPlus+ 有机分子同分异构体分析工具

## 项目简介

MolGenPlus 是一个功能强大的**有机分子同分异构体分析工具**，专为化学老师和学生设计。它可以：

- **自动生成** C1-C10 烷烃的**所有**同分异构体
- **验证** 生成的分子结构是否正确
- **筛选** 特定结构的分子（如带环的、带双键的）
- **保存** 常用分子到个人分子库

---

## 核心算法：树结构生成法

### 什么是同分异构体？

同分异构体是指**分子式相同但结构不同**的分子。例如：
- C4H10（丁烷）有两种：正丁烷和异丁烷
- C5H12（戊烷）有三种

随着碳原子数增加，异构体数量快速增长：
| 碳数 | 异构体数量 |
|------|-----------|
| C4  | 2 种      |
| C5  | 3 种      |
| C6  | 5 种      |
| C7  | 9 种      |
| C8  | 18 种     |
| C9  | 35 种     |
| C10 | 75 种     |

### 算法核心思想

本项目采用**树结构枚举算法**生成烷烃异构体。其核心思想是：

> **将烷烃分子看作一棵树**
> - 每个碳原子 = 树的节点
> - C-C 键 = 连接节点的边
> - 树的根 = 分子的"重心"（质心）

#### 举例说明

以 C4H10（丁烷）为例：

```
正丁烷：CH3-CH2-CH2-CH3
        ↓ 转化为树结构
           C(根)
          /|\
         C C C
        （4个碳连成一条链）

异丁烷：    CH3
           |
       CH3-CH-CH3
        ↓ 转化为树结构
           C(根)
          /|\
         C C C
        （3个碳连成链，1个碳作为支链）
```

### 算法步骤

#### 第一步：拆分碳原子

对于 n 个碳原子，我们将其拆分为若干"分支"：
- 每个碳最多连接 4 个其他碳（sp3 杂化）
- 所以根节点的分支最多 4 个

**拆分示例**（C4 = 4个碳）：
- 拆分为 `[3]` → 3个碳作为一条分支 → 正丁烷
- 拆分为 `[2, 1]` → 2个碳一个分支，1个碳一个分支 → 异丁烷

#### 第二步：递归生成子树

对每个拆分，递归生成子树结构：

```python
# 伪代码示例
def 生成树(碳数):
    if 碳数 == 1:
        return "C()"  # 单个碳原子
    
    # 将 碳数-1 拆分为若干部分
    for 拆分 in 所有拆分方式:
        # 递归生成每个分支的子树
        子树列表 = [生成树(分支大小) for 分支 in 拆分]
        # 组合成完整的树
        return "C(" + 子树列表 + ")"
```

#### 第三步：质心过滤（关键！）

**为什么需要过滤？**

如果我们不加过滤，同一棵树的"根"放在不同位置会生成重复的分子！

```
同一棵树，不同的根：
    C          C
   /|\        /|\
  C C C  和  C C C  是同一种分子！

选择质心作为根，可以避免重复：
    质心是树的"中心点"，每个分支不超过总数的一半
```

**过滤条件**：`每个分支的碳数 ≤ 总碳数 ÷ 2`

例如 C4：
- 拆分 `[3]`：最大分支 = 3，4÷2=2，3>2 ❌ 过滤掉
- 拆分 `[2,1]`：最大分支 = 2，4÷2=2，2≤2 ✓ 保留

#### 第四步：转换为 SMILES 格式

树结构需要转换为计算机能理解的 SMILES 字符串格式：

```
树结构：  C(根)
        /|\
       C C C

SMILES：  C(C)(C)C  （代表 2-甲基丙烷 / 异丁烷）
```

转换逻辑：
1. 从根节点开始
2. 每个分支用括号 `()` 表示
3. 递归处理所有子树

#### 第五步：RDKit 规范化去重

最后使用 RDKit 库进行规范化处理：
- 将分子转换为标准格式
- 去除重复的分子
- 验证分子式是否正确（CnH2n+2）

3. 程序会自动生成所有异构体并显示

## 技术细节（可选阅读）

### 为什么算法能生成所有异构体？

**数学证明**：树结构 + 质心过滤 = 完备枚举

1. **每个烷烃对应唯一一棵有根树**（以质心为根）
2. **每个有根树对应唯一的拆分方式**
3. **通过递归枚举所有拆分，能生成所有可能的树**
4. **质心过滤确保每棵树只被计数一次**

### 时间复杂度

| 碳数 | 生成时间 | 异构体数 |
|------|---------|---------|
| C8   | <1秒    | 18      |
| C9   | <1秒    | 35      |
| C10  | ~2秒    | 75      |

算法使用了**记忆化缓存**，相同碳数的树结构不会重复计算。

### SMILES 格式说明

SMILES（Simplified Molecular Input Line Entry System）是一种用字符串表示分子结构的方法：

| 结构 | SMILES |
|------|--------|
| 甲烷 | C |
| 乙烷 | CC |
| 丙烷 | CCC |
| 正丁烷 | CCCC |
| 异丁烷 | CC(C)C |
| 新戊烷 | CC(C)(C)C |

括号 `()` 表示支链，例如：
- `CC(C)C` = 丙烷的一个氢被甲基取代 = 异丁烷
- `CC(C)(C)C` = 中心碳连接三个甲基 = 2,2-二甲基丙烷（新戊烷）

## 常见问题

### Q: 为什么 C10 生成 75 个而不是更多？

**A**：这是化学上已证明的正确答案！C10H22 烷烃确实只有 75 种异构体。本算法经过验证，与理论值完全匹配。

### Q: 程序需要联网吗？

**A**：基本功能不需要联网。AI 增强功能（如智能去重）需要网络连接。

### Q: 如何保存生成的分子？

**A**：在界面中点击"导出"按钮，可以保存为 TXT、XYZ 或 GJF 格式。
